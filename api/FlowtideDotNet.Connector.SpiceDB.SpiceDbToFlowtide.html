Remarks

The generated Plan reads SpiceDB relationship data from a named Flowtide source table and applies the permission graph rules defined in the schema to resolve which subjects have access to which resources. Register the returned plan as a queryable view using AddPlanAsView and then reference it in SQL to obtain the expanded permission set.

Methods

Convert(string, string, string, string, bool, params string[]?)

Converts a SpiceDB schema into a Flowtide Plan that, when executed, produces the effective set of subjects that have the specified relation on the given resource type.

public static Plan Convert(string schemaText, string type, string relation, string inputTypeName, bool recurseAtStopType = false, params string[]? stopAtTypes)

Parameters

schemaText string

The SpiceDB schema DSL text to parse, defining the resource types, relations, and permission rules.

type string

The resource type to compute effective permission membership for (for example, "document"). Must be defined in the schema.

relation string

The relation or permission name to expand on the resource type (for example, "view"). Must exist on the resource type specified by type.

inputTypeName string

The name of the Flowtide table that contains the SpiceDB relationship data, matching the view or table registered with the SpiceDB source connector.

recurseAtStopType bool

When true, the permission graph is expanded through the stop types themselves before halting. When false (the default), expansion stops at the boundary with stop types without recursing into them.

stopAtTypes string[]

An optional set of resource type names at which recursive expansion should stop. Useful for bounding expansion of deeply nested permission hierarchies. Every type listed must be defined in the schema.

Returns

Plan

A Plan containing the Substrait relations that compute the effective permission set for the specified type and relation.

Remarks

The root relation of the returned plan produces rows with the following columns: subject_type, subject_id, subject_relation, relation, resource_type, and resource_id.

Exceptions

ArgumentException

Thrown when type or any type in stopAtTypes is not defined in the schema.

InvalidOperationException

Thrown when relation does not exist on the resource type specified by type.
