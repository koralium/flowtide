Defines the internal compute-layer sink for data quality check failure events raised directly from compiled SQL CHECK expressions.

public interface ICheckNotificationReceiver

Remarks

This interface is the low-level counterpart to ICheckFailureListener. Where ICheckFailureListener is the user-facing observer registered on DataflowStreamBuilder, ICheckNotificationReceiver is the compute-layer target whose reference is baked into the compiled expression tree of every SQL CHECK function at stream build time. When a CHECK condition evaluates to false for a row, the generated expression calls OnCheckFailure(in string, ReadOnlySpan<KeyValuePair<string, object?>>) directly via a constant reference captured in the expression, making this a hot-path call that may occur for every failing row during stream processing.

The sole built-in implementation is StreamNotificationReceiver, which bridges this compute-layer interface to the set of registered ICheckFailureListener instances by wrapping the arguments in a stack-allocated CheckFailureNotification ref struct and dispatching it to each listener in turn.

The receiver is wired up during stream initialization through FunctionsRegister.SetCheckNotificationReceiver, which stores it in FunctionServices so that the expression compiler can retrieve it when building CHECK function expressions.

void OnCheckFailure(in string message, ReadOnlySpan<KeyValuePair<string, object?>> tags)

Parameters

message string

The human-readable failure message declared in the SQL CHECK expression, passed by read-only reference to avoid an additional pointer copy on the call stack.

tags ReadOnlySpan<KeyValuePair<string, object>>

A stack-allocated ReadOnlySpan<T> of KeyValuePair<TKey, TValue> entries providing named diagnostic context values declared as tag arguments in the SQL CHECK expression. Must not be stored or accessed after this method returns.

Remarks

This method is invoked on the hot data path and may be called for every row in a stream batch where the check fails. Implementations should be allocation-free and return as quickly as possible to avoid stalling stream throughput.

The message parameter uses the in modifier so the string reference is passed by read-only reference without copying the pointer. The tags span is stack-allocated by the compiled expression and must not be stored or accessed after this method returns.
