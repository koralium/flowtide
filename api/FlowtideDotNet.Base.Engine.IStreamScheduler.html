Defines the contract for scheduling and dispatching recurring trigger events to stream operators.

public interface IStreamScheduler

Remarks

An IStreamScheduler is configured on a DataflowStreamBuilder via WithStreamScheduler(IStreamScheduler). If no scheduler is provided, DefaultStreamScheduler is used automatically. The stream engine calls Initialize(IStreamTriggerCaller) once during StreamContext construction, supplying itself as the IStreamTriggerCaller through which trigger events are dispatched to operators.

Schedule(string, string, TimeSpan) is called each time a vertex registers a recurring trigger (for example via RegisterTrigger inside a vertex's Initialize override). All registered schedules are cleared between stream restarts by calls to RemoveSchedule(string, string) from StreamContext.ClearTriggers, ensuring that vertices re-register their triggers with the correct interval on every startup.

Implementations are free to use any timing mechanism — for example a background Timer, a hosted service, or Quartz.NET — as long as they ultimately call CallTrigger(string, object?) when a trigger is due. DefaultStreamScheduler uses a priority-queue approach driven externally by RunAsync()'s 10 ms periodic tick loop.

Methods

Initialize(IStreamTriggerCaller)

Connects this scheduler to the stream engine and prepares it to begin dispatching triggers.

void Initialize(IStreamTriggerCaller streamTriggerCaller)

Parameters

streamTriggerCaller IStreamTriggerCaller

The IStreamTriggerCaller provided by the stream engine that routes a named trigger event to all operator vertices that registered it.

Remarks

Called once by StreamContext immediately after the scheduler is assigned, before any calls to Schedule(string, string, TimeSpan). The supplied streamTriggerCaller must be stored and used by subsequent scheduling operations to dispatch trigger events to the correct operator vertices.

RemoveSchedule(string, string)

Removes a previously registered trigger schedule so that it is no longer dispatched.

Task RemoveSchedule(string triggerName, string operatorName)

Parameters

triggerName string

The name of the trigger to remove.

operatorName string

The name of the operator that owns the trigger.

Returns

Task

A task representing the asynchronous removal operation.

Remarks

Called by StreamContext.ClearTriggers before each stream restart to reset all active schedules. Vertices re-register their triggers on every startup, so implementations must honour this removal and stop dispatching the given trigger after this call returns.

Schedule(string, string, TimeSpan)

Registers a recurring trigger that the scheduler should dispatch at the specified interval.

Task Schedule(string triggerName, string operatorName, TimeSpan interval)

Parameters

triggerName string

The name identifying the trigger within the operator.

operatorName string

The name of the operator that owns the trigger.

interval TimeSpan

The recurring interval at which the trigger should fire.

Returns

Task

A task representing the asynchronous registration operation.

Remarks

Called by the stream engine each time a vertex registers a periodic trigger during its Initialize phase. The combination of operatorName and triggerName uniquely identifies the schedule entry. Implementations must ensure that CallTrigger(string, object?) is invoked with the same operatorName and triggerName each time the interval elapses.
