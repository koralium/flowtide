Defines a receiver for state change notifications raised each time the stream transitions to a new StreamStateValue.

public interface IStreamStateChangeListener

Remarks

Implementations are registered on a DataflowStreamBuilder via AddStateChangeListener(IStreamStateChangeListener) and are invoked each time the stream enters a new lifecycle state. Multiple listeners can be registered; each is called in registration order.

OnStreamStateChange(StreamStateChangeNotification) is called synchronously within StreamContext.TransitionTo before the state machine transition begins. This means the notification arrives at the moment the new StreamStateValue is recorded — for example Starting, Running, Failure, or Stopping — before any of the associated transition work, such as stopping blocks or re-initializing the stream, is performed. Typical uses include updating an external health dashboard, logging state transitions, or signalling a dependent service.

Exceptions thrown by an implementation are swallowed by the stream engine to ensure that a misbehaving listener cannot block a state transition or disrupt stream processing.

Remarks

This method is invoked synchronously before the transition work begins; implementations should avoid long-running or blocking operations to prevent delaying the stream's lifecycle progression. Access StreamName to identify the affected stream and State for the StreamStateValue the stream is transitioning into.
