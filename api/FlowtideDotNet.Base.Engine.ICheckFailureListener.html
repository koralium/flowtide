Defines a receiver for data quality check failure notifications raised by the stream.

public interface ICheckFailureListener

Remarks

Implementations are registered on a DataflowStreamBuilder via AddCheckFailureListener(ICheckFailureListener) and are invoked each time a SQL-level CHECK assertion evaluated within the stream produces a failing result. Multiple listeners can be registered; each is called in registration order.

The notification is passed as a ref readonly CheckFailureNotification, which is a ref struct. This avoids a heap allocation and allows the notification to carry a ReadOnlySpan<T> of diagnostic tags. Implementations must therefore not store the notification beyond the duration of OnCheckFailure(ref readonly CheckFailureNotification) — copy any required fields to local variables instead.

Exceptions thrown by an implementation are swallowed by the stream engine to ensure that a misbehaving listener cannot disrupt stream processing. The built-in implementations are LoggerCheckFailureListener, which forwards the failure message and tags to ILogger, and ActivityCheckFailureListener, which emits an OpenTelemetry Activity under the source FlowtideDotNet.CheckFailures.

Remarks

The notification is a ref readonly ref struct; it must not be stored or captured beyond the scope of this method call. Access StreamName to identify the originating stream, Message for the human-readable failure description, and Tags for the associated diagnostic key-value metadata.
