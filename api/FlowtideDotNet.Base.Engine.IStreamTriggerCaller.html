Defines the contract for dispatching trigger events to one or all operators within a stream.

public interface IStreamTriggerCaller

Remarks

IStreamTriggerCaller is implemented by the stream engine's StreamContext and is passed to an IStreamScheduler via Initialize(IStreamTriggerCaller) so that the scheduler can dispatch trigger events when a scheduled interval elapses. It is also surfaced publicly on CallTrigger(string, object?) for imperative, on-demand trigger invocations from outside the stream.

Two dispatch modes are available. The broadcast overload CallTrigger(string, object?) routes the trigger to every operator that has registered a handler for the given triggerName, making it suitable for stream-wide signals such as a scheduled flush. The targeted overload CallTrigger(string, string, object?) routes the trigger to a single named operator, making it suitable for precise, per-operator control. In both cases, each matching operator receives the trigger via its OnTrigger override by queuing a TriggerEvent onto the operator's internal dataflow block.

Methods

CallTrigger(string, object?)

Dispatches a named trigger to every operator in the stream that has registered a handler for triggerName.

Task CallTrigger(string triggerName, object? state)

Parameters

triggerName string

The name identifying the trigger, matching the name used when the operator registered the trigger via RegisterTrigger.

state object

An optional context object forwarded as-is to each operator's OnTrigger override. Pass null when no additional context is needed.

Returns

Task

A task that completes once the trigger event has been queued on all matching operators.

Remarks

The engine looks up all operators registered under triggerName and queues a TriggerEvent on each one. If no operator has registered the given trigger name the call is a no-op. The returned task completes once the event has been enqueued on all matching operators, not once their OnTrigger handlers have finished.

CallTrigger(string, string, object?)

Dispatches a named trigger to a single specific operator within the stream.

Task CallTrigger(string operatorName, string triggerName, object? state)

Parameters

operatorName string

The unique name of the target operator within the stream, as set during IStreamVertex.Setup.

triggerName string

The name identifying the trigger, matching the name used when the operator registered the trigger via RegisterTrigger.

state object

An optional context object forwarded as-is to the operator's OnTrigger override. Pass null when no additional context is needed.

Returns

Task

A task that completes once the trigger event has been queued on the target operator, or immediately if no matching operator or trigger registration was found.

Remarks

The engine looks up the operator identified by operatorName and, if it has registered a handler for triggerName, queues a TriggerEvent on it. If the operator does not exist or has not registered the given trigger name the call is a no-op. The returned task completes once the event has been enqueued, not once the operator's OnTrigger handler has finished.
