Remarks

Scheduling is backed by a min-heap priority queue keyed on the next UTC fire time, together with a dictionary that maps each {operatorName}.{triggerName} key to its recurrence interval. When Schedule(string, string, TimeSpan) is called, the trigger's first fire time is set to DateTime.UtcNow + interval and the entry is enqueued. On each Tick() call, all entries whose fire time is in the past are dequeued, re-enqueued at the next interval, and dispatched via the IStreamTriggerCaller supplied at Initialize(IStreamTriggerCaller).

Tick() is not called automatically; it must be driven externally. When using RunAsync(), the stream's internal 10 ms periodic loop calls Tick() on every iteration. Custom IStreamScheduler implementations or hosting scenarios that call StartAsync() directly are responsible for driving Tick() at the desired resolution.

All public members are declared virtual so that derived classes can override individual scheduling behaviours without replacing the entire implementation.

Remarks

The trigger is enqueued immediately with a first fire time of DateTime.UtcNow + interval. On each subsequent fire, it is automatically re-enqueued at DateTime.UtcNow + interval from the moment it was dispatched. The combined key {operatorName}.{triggerName} must be unique; registering a duplicate key will throw a ArgumentException from the underlying dictionary.

Tick()

Checks the schedule queue and dispatches all triggers whose fire time has passed.

public virtual Task Tick()

Returns

Task

A task that completes when all overdue triggers for this tick have been dispatched.

Remarks

Each overdue trigger is dequeued, re-enqueued at DateTime.UtcNow + interval to schedule its next recurrence, and then dispatched via CallTrigger(string, object?). If the trigger has been removed via RemoveSchedule(string, string) since it was enqueued, it is silently discarded without being dispatched.

The lock is released around each CallTrigger invocation so that scheduling operations from other threads can proceed while the trigger handler executes.

This method must be called externally to advance the scheduler. When using RunAsync(), it is called automatically inside the 10 ms timer loop.
