Remarks

This class implements both IAsyncEnumerable<T> and IAsyncEnumerator<T> on the same instance to avoid a secondary heap allocation for the enumerator. As a result, GetAsyncEnumerator(CancellationToken) returns this and the class is not safe for concurrent enumeration and must not be enumerated more than once.

It is used throughout the dataflow vertex pipeline wherever a method returning IAsyncEnumerable<T> must produce exactly one output item, for example when a checkpoint handler or a single-result OnRecieve override needs to emit a single IStreamEvent without the overhead of a full iterator state machine. For the zero-element counterpart see EmptyAsyncEnumerable<T>.
