Remarks

Each instance wraps a lazily initialized FlowtideDotNet.Base.Vertices.MultipleInputTarget<T> and is identified by a zero-based targetId that determines which input index is being fed when messages arrive at the parent vertex. The internal FlowtideDotNet.Base.Vertices.MultipleInputTarget<T> is created during the vertex setup phase via Initialize, and the holder must be fully set up (via Setup) before the OperatorName and Completion members are accessible. Upstream dataflow blocks are connected to this holder using standard TPL Dataflow link mechanics, which the holder forwards to the underlying target block.
