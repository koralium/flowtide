Represents the high-level lifecycle state of a Flowtide dataflow stream.

public enum StreamStateValue

Fields

Deleted = 5

All persistent state associated with the stream has been permanently removed from storage. This is a terminal state; StopAsync() throws NotSupportedException in this state. Corresponds to Deleted.

Deleting = 4

The stream is permanently removing all operator state and persisted data from storage. All dataflow blocks are completed, awaited, disposed, and individually deleted before the stream transitions to Deleted. Corresponds to Deleting.

Failure = 3

An unhandled exception has occurred. All running blocks are stopped and disposed, the stream rolls back to the last committed checkpoint, and a restart is automatically scheduled by transitioning back to Starting. This enables automatic self-healing without external intervention. Corresponds to Failing.

NotStarted = 0

The stream has not been started, or has returned to an idle state after a graceful stop. This is the initial value assigned when a stream is built via DataflowStreamBuilder, and the state that the stopping sequence transitions back to once all blocks have been completed and disposed. RunAsync() exits when this state is reached. Corresponds to Stopped.

Running = 2

The stream is fully initialized and is actively processing data through its operator graph. Triggers are active, checkpoints can be taken, and data flows through all connected vertices. The stream exits this state on a graceful stop request (Stopping), a delete request (Deleting), or an unhandled exception (Failure). Corresponds to Running.

Starting = 1

The stream is actively initializing its operator graph, linking dataflow blocks, and restoring operator state from the last committed checkpoint. The stream remains in this state until all egress vertices have completed their initial watermark handshake, at which point it transitions to Running. Any unhandled exception during startup transitions the stream to Failure. Corresponds to Starting.

Stopping = 6

A graceful shutdown is in progress. The stream triggers a final checkpoint across all egress vertices, then completes and disposes all blocks before transitioning back to NotStarted. StopAsync() awaits this entire sequence before returning. Corresponds to Stopping.

Remarks

StreamStateValue tracks the current position within the stream's lifecycle state machine, complementing the finer-grained StreamStatus which reflects the operational condition at a more detailed level. Where StreamStatus surfaces transient conditions such as pausing or degradation, StreamStateValue conveys the coarse lifecycle phase — for example whether the stream is starting up, running normally, recovering from a failure, or shutting down.

The current value is exposed via State. Components that need to react to lifecycle transitions can implement IStreamStateChangeListener and register via AddStateChangeListener(IStreamStateChangeListener). Each transition fires all registered listeners synchronously before the associated state machine work begins.
