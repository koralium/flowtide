Remarks

Register this strategy via AddFailureListener(IFailureListener) or use FlowtideBuilder.WithFailureListener(Action<Exception?>), which constructs a CustomExceptionStrategy internally, to react to stream failures without implementing the full IFailureListener interface.

Unlike ExitProcessStrategy and KillProcessStrategy, this strategy does not terminate the process. After the callback returns, the stream's built-in automatic recovery sequence continues normally, restarting the stream from the last committed checkpoint. Typical uses for the callback include custom logging, sending an alert, incrementing a failure metric, or signalling an external monitoring system.

public void OnFailure(StreamFailureNotification notification)

Parameters

notification StreamFailureNotification

The StreamFailureNotification describing the stream failure. Only Exception is forwarded to the callback; StreamName is available if a custom IFailureListener implementation is preferred over this strategy.

Remarks

The callback receives Exception, which may be null if no specific exception was captured. After the callback returns, the stream's built-in automatic recovery proceeds normally; the callback must not block indefinitely, as this method is invoked synchronously within the failure transition.
