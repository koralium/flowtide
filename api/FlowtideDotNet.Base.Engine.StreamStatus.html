Represents the fine-grained operational status of a Flowtide dataflow stream.

public enum StreamStatus

Fields

Degraded = 3

The stream is operational but experiencing reduced capacity or a transient non-fatal condition, for example database locks from sources. Maps to Degraded.

Deleted = 7

All persistent state associated with the stream has been successfully deleted. Set when DeletedStreamState initialization completes. Maps to Unhealthy.

Deleting = 6

The stream is permanently removing all operator state from persistent storage. Set during DeletingStreamState initialization. Maps to Unhealthy.

Failing = 4

An unhandled exception has occurred. The stream is stopping all running blocks, rolling back to the last committed checkpoint, and preparing to restart automatically. Set during FailureStreamState initialization. Maps to Unhealthy.

Paused = 8

Message processing is temporarily suspended via Pause(). Data continues to queue in operator buffers but is not processed until Resume() is called. Maps to Healthy when the underlying StreamStateValue is Running, Unhealthy when it is Failure, and Degraded otherwise.

Running = 2

The stream is fully initialized and is actively processing data through its operator graph. Set when the RunningStreamState transition completes successfully. Maps to Healthy.

Starting = 1

The stream is initializing its dataflow blocks, linking vertices, and restoring operator state from the last committed checkpoint. Set during the StartStreamState transition. Maps to Degraded.

Stopped = 0

The stream has not yet been started or has completed a graceful stop. This is the initial status before StartAsync() is called and the terminal status after a graceful stop sequence completes. Maps to Unhealthy.

Stopping = 5

The stream is executing a graceful shutdown sequence, taking a final FlowtideDotNet.Base.StopStreamCheckpoint across all egress vertices before completing. Set during StoppingStreamState initialization. Maps to Unhealthy.

Remarks

StreamStatus provides more granular operational information than StreamStateValue, which tracks the high-level lifecycle state machine position. Where StreamStateValue reflects the state machine transition (e.g. Starting, Running, Failure), StreamStatus reflects what the stream is actively doing at an operational level, including transient states such as pausing and graceful stopping.

The current status is exposed via Status and is also used to derive the FlowtideHealth value reported by Health and the flowtide_status metric gauge. Both StreamStatus and FlowtideHealth are published as observable gauges on the flowtide.{streamName} meter for integration with external monitoring systems.
