Represents a filter on the subject side of a relationship in SpiceDB.

public interface ISpiceDbSubjectFilter

Remarks

This interface wraps the Authzed API v1 SubjectFilter message. SubjectType is required; all other properties are optional and impose no additional constraint when left unset. A subject filter is supplied as OptionalSubjectFilter to narrow a relationship query by the type, identifier, or sub-relation of its subject side. Use the static Create(string?, string?, ISpiceDbRelationFilter?) factory method to construct a new instance.

When not set, subjects of any identifier matching the given type are returned. Supports the wildcard value "*" to match all subjects of the given type.

Parameters

subjectType string

The type of the subject to filter on, as defined in the SpiceDB schema (for example, "user"). If null, the subject type will not be set.

optionalSubjectId string

The specific subject identifier to match. If null, the subject identifier filter will not be applied.

optionalRelation ISpiceDbRelationFilter

An ISpiceDbRelationFilter to further narrow results by the sub-relation held by the subject. If null, no sub-relation filter will be applied.
