Represents the consistency requirements for a SpiceDB API request, controlling how data freshness is balanced against latency.

public interface ISpiceDbConsistency

Remarks

This interface wraps the Authzed API v1 Consistency message. Each request must specify exactly one consistency requirement. Use the static factory methods (CreateFullyConsistent(), CreateMinimizeLatency(), CreateAtExactSnapshot(ISpiceDbZedToken), CreateAtLeastAsFresh(ISpiceDbZedToken)) to create instances.

Properties

AtExactSnapshot

Gets or sets the ISpiceDbZedToken that specifies the exact snapshot at which all data used in the API call must be evaluated.

ISpiceDbZedToken AtExactSnapshot { get; set; }

Property Value

ISpiceDbZedToken

Remarks

If the specified snapshot is no longer available, an error will be returned to the caller.

AtLeastAsFresh

Gets or sets the ISpiceDbZedToken that specifies the minimum freshness of data used in the API call.

ISpiceDbZedToken AtLeastAsFresh { get; set; }

Property Value

ISpiceDbZedToken

Remarks

All data used in the API call must be at least as fresh as that found in the provided ISpiceDbZedToken. More recent data may be used if it is available or faster.

FullyConsistent

Gets or sets a value indicating that all data used in the API call must be at the most recent snapshot found.

bool FullyConsistent { get; set; }

Property Value

bool

Remarks

Using this option can be quite slow. Unless there is a specific need to enforce full consistency, it is recommended to use AtLeastAsFresh with a stored ISpiceDbZedToken instead.

HasFullyConsistent

Gets a value indicating whether FullyConsistent has been set on this instance.

bool HasFullyConsistent { get; }

bool
