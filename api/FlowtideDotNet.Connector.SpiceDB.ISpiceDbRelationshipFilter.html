Represents a collection of filters that, when applied to a set of relationships, return only those with exactly matching fields.

public interface ISpiceDbRelationshipFilter

Remarks

This interface wraps the Authzed API v1 RelationshipFilter message. ResourceType is required; all other properties are optional and impose no additional constraint when left unset. A filter is supplied as RelationshipFilter to scope a read operation, and as Filter to define the relationship set evaluated by a precondition. Use the static Create(string?, string?, string?, ISpiceDbSubjectFilter?) factory method to construct a new instance.

Parameters

resourceType string

The type of the resource to filter on, as defined in the SpiceDB schema (for example, "document"). If null, the resource type will not be set.

optionalResourceId string

The specific resource identifier to match. If null, the resource identifier filter will not be applied.

optionalRelation string

The relation name to match (for example, "reader"). If null, the relation filter will not be applied.

optionalSubjectFilter ISpiceDbSubjectFilter

An ISpiceDbSubjectFilter to further narrow results by subject type, identifier, or sub-relation. If null, no subject filter will be applied.

Returns

ISpiceDbRelationshipFilter

A new ISpiceDbRelationshipFilter with the specified resource type and narrowing criteria.
