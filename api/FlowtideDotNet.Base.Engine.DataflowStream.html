Remarks

A DataflowStream wraps an internal StreamContext state machine and exposes an API for starting, stopping, pausing, checkpointing, and inspecting the stream. Instances are created via Build() and are not intended to be constructed directly.

The stream can be driven in two ways depending on the configured IStreamScheduler:

  • Call RunAsync() to start the stream and block the caller within a scheduler tick loop. This is the typical usage when using the DefaultStreamScheduler, and is how the hosted service StreamWorkerService operates it.
  • Call StartAsync() to initiate the startup sequence only, leaving scheduler tick management to the caller. This is appropriate for custom IStreamScheduler implementations.

The stream progresses through StreamStateValue states (e.g. NotStarted, Starting, Running, Failure) and StreamStatus operational statuses (e.g. Running, Paused, Failing). The combined Health property derives a FlowtideHealth value suitable for health checks and monitoring.

RunAsync()

Starts the stream and drives the scheduler tick loop until the stream transitions to NotStarted.

public Task RunAsync()

Returns

Task

A task that completes when the stream stops running.

Remarks

This method is only active when the configured IStreamScheduler is a DefaultStreamScheduler. It calls StartAsync() internally, then enters a 10 ms periodic timer loop that calls Tick on the scheduler to dispatch any due trigger events.

A full garbage collection is performed once every 10 seconds (every 1000 ticks) to manage native memory pressure from large stream workloads.

When using a custom IStreamScheduler, use StartAsync() directly instead and manage the scheduler externally.
