Remarks

DataflowStreamBuilder assembles all components required to run a dataflow stream: ingress vertices (sources), propagator vertices (transforms), and egress vertices (sinks), together with state management, scheduling, logging, versioning, and notification callbacks. Each configuration method returns the same builder instance to support method chaining.

The only required configuration before calling Build() is WithStateOptions(StateManagerOptions), which provides the state manager persistence settings. All other configuration is optional and falls back to sensible defaults:

  • WithStreamScheduler(IStreamScheduler) defaults to DefaultStreamScheduler.
  • WithStateHandler(IStateHandler) defaults to a no-op NullStateHandler.

For higher-level SQL-plan-based stream construction, prefer FlowtideBuilder from FlowtideDotNet.Core, which wraps this builder and handles plan compilation, operator wiring, and hash-based version tracking automatically.

Build()

Validates the configuration, applies defaults for any unset optional components, and constructs the fully configured DataflowStream.

public DataflowStream Build()

Returns

DataflowStream

A fully configured but not yet started DataflowStream.

Remarks

If WithStateHandler(IStateHandler) was not called, a no-op NullStateHandler is used. If WithStreamScheduler(IStreamScheduler) was not called, a DefaultStreamScheduler is created.

The returned DataflowStream is not yet started. Call StartAsync() or RunAsync() to begin processing.

Exceptions

InvalidOperationException

Thrown if WithStateOptions(StateManagerOptions) has not been called before Build().
