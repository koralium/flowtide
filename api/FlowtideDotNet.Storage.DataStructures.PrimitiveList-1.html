Class PrimitiveList<T>

Namespace
FlowtideDotNet.Storage.DataStructures

A growable list of unmanaged values stored in native memory allocated from an IMemoryAllocator. It behaves like a list (add, insert, remove, index) but keeps its elements in a contiguous unmanaged buffer, which can be exposed as a Span or Memory and serialized without copying element by element. Ownership is reference counted through Rent(int) and Return(), and the buffer is released when the list is disposed or the last reference is returned.

InsertFrom(ref readonly PrimitiveList<T>, ref readonly ReadOnlySpan<int>, ref readonly ReadOnlySpan<int>, in int)

Special case insert that allows inserting a subset of elements from another primitive list at specific positions. This is used when merging two lists together more memory efficiently than inserting each element one by one. The sortedLookup and insertPositions spans must have the same length, but do not need to cover every element in the other list. The positions in insertPositions are interpreted relative to the original contents of the current list before any elements are inserted. Conceptually, this behaves like inserting the selected elements in order using InsertAt(insertPositions[i] + i, ...).
