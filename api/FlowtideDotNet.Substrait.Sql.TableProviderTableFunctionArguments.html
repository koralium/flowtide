Remarks

An instance is created by the SQL visitor for every table-valued function reference it encounters — either as a standalone FROM source or as the right-hand side of a JOIN. When used in a JOIN, ParentRelation, JoinType, and JoinCondition are populated so the implementation can build a joined relation directly; all three are null when the function is used as a standalone FROM source.

Implementations of TryHandleTableFunction(IReadOnlyList<string>, TableProviderTableFunctionArguments, out TableProviderTableFunctionResult?) receive this object and use Arguments, ExpressionVisitor, and EmitData to evaluate the individual SQL arguments and resolve column references. JoinType should be checked first when the function does not support being placed inside a JOIN.

Remarks

When the function appears as a standalone FROM source this is an empty EmitData instance. When called from a JOIN it contains the columns contributed by the left-hand ParentRelation, allowing argument expressions to reference columns from the left side of the join.
